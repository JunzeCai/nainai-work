# Group Chat Workflow

Build this internally:

```yaml
beat_sheet:
  opening_heat: ""
  wrong_take: ""
  correction: ""
  conflict_points: []
  life_scene: ""
  change_of_mind: ""
  ending_action: ""
```

## Rhythm

1. Line 1-5: quick heat. Someone makes a claim, asks a blunt question, or throws in a screenshot/news angle.
2. Line 6-15: disagreement and confusion.
3. Line 16-28: someone corrects details; another person relates it to dinner, gifting, budget, or channel.
4. Final lines: practical but not summary-like.

## Novelty Loop

When the user asks for another version in the same session, create a `difference_contract` first:

- different opening trigger
- different main conflict
- different roles leading the conversation
- different ending action
- different life scene

Do not merely swap synonyms.
